# Android Academic Planner App

The Android Academic Planner App is a mobile application designed to help students keep track of their academic schedules. The app allows users to create and manage multiple academic terms, add courses to those terms, and track assessments for each course.

## Functional Requirements

The Android Academic Planner App includes the following functional requirements:

1. Users can create academic terms, including a title, start date, and end date.
2. Users can add multiple terms to the app.
3. The app will prevent users from deleting a term if courses are assigned to it.
4. Users can add courses to each term, including a title, start date, end date, status (in progress, completed, dropped, or plan to take), and instructor information.
5. Users can add assessments to each course, including a title and end date.
6. Users can set alerts for course start and end dates, as well as assessment start and end dates.
7. Users can share notes for a course via email or SMS.

## Screen Layouts

The Android Academic Planner App includes the following screen layouts:

1. Home screen: displays a list of terms.
2. List of terms: displays a list of all academic terms added to the app.
3. List of courses: displays a list of all courses added to the selected term.
4. List of assessments: displays a list of all assessments added to the selected course.
5. Detailed course view: displays the details for the selected course, including start date, end date, status, and instructor information.
6. Detailed term view: displays the details for the selected term, including start date, end date, and a list of courses.
7. Detailed assessment view: displays the details for the selected assessment, including title and end date.

## Scheduler

The Android Academic Planner App includes a scheduler that provides the following elements:

1. An ArrayList to store application data.
2. An intent to navigate between activities.
3. Navigation capability between multiple screens using activities.
4. Three activities: TermListActivity, CourseListActivity, and AssessmentListActivity.
5. Events such as a click event.
6. The ability to work in both portrait and landscape layout.
7. Interactive capability, such as accepting and acting on user input.
8. A database to store and retrieve application data.
9. An application title and icon.
10. Notifications or alerts.
11. Declarative and programmatic methods to create a user interface.

## Interface Requirements

The Android Academic Planner App meets the following interface requirements:

1. The ability to scroll vertically to see all course and assessment details.
2. An action bar to provide easy navigation.
3. Two layouts, one for the list views and one for the detailed views.
4. Input controls, such as text fields and date pickers, to allow users to enter information.
5. Buttons to trigger events, such as adding a new course or assessment.

## Future Enhancements

Future enhancements to the Android Academic Planner App could include:

1. Integration with external calendars, such as Google Calendar or Microsoft Outlook.
2. Adding support for recurring assessments, such as weekly quizzes or monthly exams.
3. Incorporating a grade tracker to monitor course grades throughout the term.
4. Adding the ability to import course schedules from external sources, such as university websites.
5. Including a feature to track attendance for each course.
